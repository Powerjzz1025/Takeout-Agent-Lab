# Step 6：RAG 知识库检索

这一阶段的目标是把 Step 5 中的 `rag_lookup` 路由变成真正可用的知识库查询能力。

当前实现是本地关键词检索版 RAG，用于学习和 Demo。后续可以替换成 embedding、向量数据库和大模型总结。

## 为什么要做 RAG

外卖 Agent 不能只依赖菜单标签。很多用户需求是知识型或经验型：

- 胃不舒服适合吃什么？
- 减脂外卖怎么点？
- 20 分钟内配送应该选什么类型的店？
- 川菜是不是都很辣？
- 花生过敏下单要注意什么？

这些不是简单搜索餐厅能解决的，需要查询知识库。

## 本步新增文件

### `data/knowledge-base.json`

核心意图：建立外卖点餐知识库。

用于做什么：存放可被检索的饮食知识、点餐策略和安全提醒。

当前包含：

- 胃不舒服时的外卖选择
- 减脂期外卖点餐策略
- 清淡工作餐选择
- 高蛋白点餐策略
- 川菜点餐注意事项
- 常见过敏与忌口提醒
- 20 分钟内配送策略
- 30 到 40 元预算策略

产品经理理解：这是 Agent 的“领域知识”。菜单数据告诉系统有什么，知识库告诉系统为什么这样推荐。

### `rag.js`

核心意图：建立 RAG 检索器。

用于做什么：

- 接收 query 和槽位
- 生成检索关键词
- 在知识库中打分
- 返回命中的知识片段
- 生成一个简短回答

Agent 架构位置：Retrieval / Knowledge Layer。

未来接入：向量数据库、embedding、混合检索、大模型摘要。

## 本步关键函数

### `createRagRuntime()`

核心意图：创建知识库检索运行时。

用于做什么：接收知识库数据，返回 `retrieve()` 方法。

产品经理理解：这是 RAG 能力的入口，相当于“知识库查询服务”。

### `retrieve()`

核心意图：执行一次知识检索。

用于做什么：根据用户问题和槽位，从知识库里找最相关的内容。

产品经理理解：这是 RAG 的 R，也就是 Retrieval。

### `buildQueryTerms()`

核心意图：生成检索词。

用于做什么：把用户 query、意图槽位、健康目标、菜系、口味和忌口合并成检索关键词。

产品经理理解：同一句话里，不是所有词都重要。槽位能帮助系统抓住关键检索词。

### `scoreDocument()`

核心意图：给知识片段打分。

用于做什么：命中标签、标题、摘要、推荐项时加不同分数。

产品经理理解：这是当前 Demo 的简化检索算法。真实 RAG 会用 embedding 计算语义相似度。

### `buildAnswer()`

核心意图：基于命中的知识片段生成回答。

用于做什么：把知识库摘要、推荐项、避雷项拼成用户可读回答。

产品经理理解：真实系统里，这一步通常会交给大模型做总结和措辞。

## 本步更新的已有模块

### `tools.js`

新增工具：

- `retrieveFoodKnowledge`

目的：让 RAG 成为 Agent 可调用工具。

产品经理理解：RAG 不应该散落在 Workflow 里，而应该和搜餐厅、查菜单一样，成为一个标准工具。

### `workflow.js`

更新点：

- `rag_lookup` 路由不再是占位，而是调用 `retrieve_food_knowledge`
- 标准点餐推荐前，也会检索一次知识库，用于辅助解释

产品经理理解：知识库既能单独回答问题，也能参与推荐理由。

### `app.js`

新增状态：

- `KnowledgeResults`

目的：展示 RAG 检索结果、命中片段和生成回答。

产品经理理解：做 RAG 产品一定要能看见“命中了什么资料”，否则无法判断回答是否可靠。

## 当前 RAG 链路

```text
用户 query
-> IntentResult 判断 route = rag_lookup
-> retrieve_food_knowledge 工具
-> rag.js 检索 knowledge-base.json
-> 返回 KnowledgeResults
-> Workflow 生成回答
```

点餐推荐链路中也会查询知识库：

```text
点餐需求
-> 检索相关饮食知识
-> 搜餐厅和菜单
-> 排序菜品
-> 推荐时引用知识库解释
```

## 当前局限

这是关键词版 RAG，不是最终形态。

局限：

- 不能理解复杂语义相似
- 知识库片段很少
- 回答是模板拼接，不是大模型总结
- 没有引用来源展示和置信度校验

后续升级：

- 把知识片段切 chunk
- 用 embedding 生成向量
- 接入向量数据库
- 做 topK 检索
- 把命中片段交给大模型生成回答
- 在页面展示引用片段
